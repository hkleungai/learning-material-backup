Q:

 Can we set the server to schedule a specific task to be carried out at a specific time, such as instruct the server to run a certain page at a certain time?

A:

While it is possible in ASP.NET to schedule a task to run at a specific time, to do so requires administrator rights on the computer on which the task is running. Since you will do the testing in Lab 4, you do not have such rights on those computers. Therefore, you cannot use this function of ASP.NET.

However, you can create threads that run concurrently and that can be awakened periodically to run tasks. To create a thread that runs periodically, do the following in the "Global.asax.cs" file.

  1. Add the statement "using System.Threading;" at the top of the file.
  2. Place the following code inside the "Application_Start" method.
    3. Thread mythread = new Thread(PeriodicTasks);
mythread.IsBackground = true;
mythread.Start();
  3. Place the following method inside "public class Global : HttpApplication" after the "Application_Start" method.
private void PeriodicTasks()
{
  do
  {
    // Place the method call for the periodic task here.
    Thread.Sleep(time-in-milliseconds);
  } while (true);
}


Email Sending

Q:

How can we send email from our system?

A:

You can send email directly from a program in ASP.NET.

First, place the following code in the Web.config file. 

<system.net>
  <mailSettings>
    <smtp deliveryMethod="Network" from="<team-email-address>@cse.ust.hk">
      <network defaultCredentials="false" host="smtp.cse.ust.hk" enableSsl="true" 
       port="587" userName="<team-email-address>@cse.ust.hk" 
       password="<team-email-password>" />
    </smtp>
  </mailSettings>
</system.net>

Then, in Visual C# use the following code. 

// Add namespaces to allow use of the mail sending functionality.
using System.Net;
using System.Net.Mail;
// Create an instance of MailMessage named mail.
MailMessage mail = new MailMessage();
   
// Create an instance of SmtpClient named emailServer.
// Set the mail server to use as "smtp.ust.hk".
SmtpClient emailServer = new SmtpClient("smtp.ust.hk");
// Set the sender (From), receiver (To), subject and 
// message body fields of the mail message.
mail.From = new MailAddress("<team-email-address>@cse.ust.hk", 
     "<sender name>");
mail.To.Add("<team-email-address>@cse.ust.hk");
mail.Subject = "<subject>";
mail.Body = "<message body>"; 
// Send the message.
emailServer.Send(mail);

-------------------------------------

When testing the above code, be sure to replace all items in <angle brackets and red font> with the values you want to use.

EXTREMELY IMPORTANT! You must use valid email addresses in the recipient list (i.e., mail.To.Add()). If you specify invalid/non-existent recipient email addresses, then the CSE mail server will detect this, consider you are trying to send junk messages and automatically block your email address from sending further email.

Q:

Must we use the UST email server to send email? Can we use other email servers like the gmail email server?

A:

Only the CSE (smtp.cse.ust.hk) or ITSC (smtp.ust.hk) mail servers can be used to send mail from the computer in Lab 4. The CSE mail server can only be used using your team email accounts.

Q:

How can we read the email sent using our team email account?

A:

The CSE mail server is an imap mail server, so you can set up an imap email account in your favourite email client using the team email account's user name and password.

You can also use CSE WebMail (https://imail.cse.ust.hk/sqmail/src/login.php) using your team email acount's user name and password.
