General information

  • Release date: 24th Mar, 2015
  • Description: Comp2611Project2015Spring.pdf
  • Mars4.4_withSyscall100.jar
    • This Mars is a specifically version made for this project.
    • All the syscalls that are needed in this project are documented in the description file above.
  • Programming files (daleks.s, background.gif, drwho1.png, drwho2.png, dalek.png, rubble.png): daleks.zip 
    • unzip all the files into a same directory, download our Mars4.4 and put it to the same directory (otherwise errors could occur).
    • Add your code to the skeleton file (daleks.s), the skeleton file provides a starting point for you.
    • Read the skeleton file carefully to further understand the programming tasks.
  • Project demo video: DaleksGameDemo.mp4
  • Start as early as possible!
  • Notes:
    • 1. Drwho dies when he walks on the rubbles.
    • 2. Screwdriver makes the daleks in the range disappear (instead of become rubbles).
    • 3. In diagonal moves (press '1','3','7','9'), Drwho moves in each step 10 pixels in both of the two directions.
    • 4. The Screwdriver does not accumulate to the next level.
    • 5. Pressing 'r' after the Screwdriver is used up should be treated as an invalid input, and the daleks will not move.
    • 6. The effective range of the Screwdriver covers a square whose edge length is 120 pixels,and the center of the rectangle is Drwho.
    • 7. The rubbles will not be removed by the Screwdriver.
