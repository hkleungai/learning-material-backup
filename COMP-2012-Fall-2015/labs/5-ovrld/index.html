Basic class Syntax

·        A class consists of members. The members can be either data or functions.

·        The functions are called member functions (methods).

·        Each instance of a class is an object.

·        Each object contains the data components specified in the class.

·        A member function is used to act on an object.

 

Container class

About container classes: container.pdf

·        Classes designed to hold collections of objects

·        Commonly provided services such as insertion, deletion, searching, sorting and testing an item to determine whether it is a member of the collection.

 

Example:

class PersonContainer

{

   public:

      ...

 

   private:

      Person* array;

      ...

}          

 

The operations on container classes can be:

  • member functions of the container class
  • global functions that take a container argument

 

Three concepts of container class implementation:

  • the kind of container (e.g., list-based, array-based)
  • the kind of objects stored in the container (e.g., Person, int)
  • the kind of operations on the elements stored in the container (i.e., "do something for each element")

 

Usually, there is a strong coupling between the three concepts.

Other examples of containers: Arrays, Linked Lists, Stacks, Queues, Trees

 

An Example: The Bag class

Consider a bag for storing numbers, the following is a class definition of it:

class bag

{

    public:

        // typedefs & Member constants

        typedef int value_type;

        typedef std::size_t size_type;

        static const size_type CAPACITY = 30;

 

        // constructor

        bag( ) { used = 0; }

 

        // modifying member functions

        size_type erase(const value_type& target);

        bool erase_one(const value_type& target);

        void insert(const value_type& entry);

        void operator +=(const bag& addend);

 

        // constant member functions

        size_type size( ) const { return used; }

        size_type count(const value_type& target) const;

 

    private:

        value_type data[CAPACITY];  // The array to store items

        size_type used;             // How much of array is used

};

 

The source files of the bag class:

   bag.h, a header file that contains

   documentation

   class definition

   bag.cpp, an implementation file that contains

   the implementations of the bag’s member functions

   driver.cpp, the main program using the bag class

 

 

Another Example: Array Implementation of Lists    

 

A list is a linear sequence of objects and can be implemented by arrays.

The following is linear list class implemented using a static array.

 

class List

{

   private:

      // member variables

      int length;

int element[3];

    

   public:

      // constructor(s) for initialization

      List(int size);

 

      // destructor(s)

      ~List();

 

      // member methods

      int isEmpty() const {return length==0;}

         // return 0 if the list is empty, i.e. contains no elements

 

      int Find(int k,int &x) const;

         // return 1 if there exist an element at position k, store the value in x. else return 0.

 

      int Search( int x) const;

         // return 0 if x does not exist, else return the position of x in the list.

 

      void Delete(int k);

         // Delete the element at position k.

 

      void Insert(int k, int x);

         // Insert a value to position k in the list

 

      void Output() const;

         // Print the elements of the list

};

 

 

Friend Function

Private and protected members of a class cannot be accessed outside the class in which they are declared.

However, a non-member function can access the private and protected members of a class if it is declared a friend of it.

This is done by including a declaration of the non-member function within the class and preceding it with the friend keyword.

 

For example:

 

class X {

  private:

    int data;

  public:

    X() {}

    X(int x) : data(x) {}

int getData() const { return data; }

 

friend int fn(const X& x, const X& y);

};

 

int fn(const X& x, const X& y)

{

   return (x.data+y.data);

}

Note: Similar to friend function, a friend class can access the private or protected members of another class.

 

Operator Overloading

A few points to be noted:

  • After operator overloading, we can use operators with objects
  • Almost all operators in C++ can be overloaded. Four exceptions:
    .     :: ?: sizeof
  • We can only redefine existing operators, but CANNOT define new operators
  • We CANNOT change the properties of an operator. (Number of arguments an operator takes, associativity and precedence).
