Overview of Qt

Qt is a C++ toolkit for multiplatform GUI and application development. In addition to the C++ class library, Qt includes tools to make writing applications fast and straightforward.
Executive summary:
a rich set of widgets that provide standard GUI functionality
a variety of domain-specific classes
excellent support for 2D and 3D graphics
can create platform-independent database applications
Qt applications can have their functionality extended by plugins and dynamic libraries.
Qt is a mature C++ toolkit that is widely used across the world.
Widgets are visual elements that are combined to create user interfaces, e.g., buttons, menus, scroll bars, message boxes, application windows and so on.

 

Setup Qt at CSD UG domain

All QT lab exercises and the sample project 2 tetris were compiled using QT 3.2. To compile using QT 3.2, add the following alias in your ~/.cshrc_user:
alias qmake /usr/lib/qt-3.3/bin/qmake
To define the required environment variables for Qt, add also the following lines to your ~/.cshrc_user , or you can simply run the command.
setenv QMAKESPEC linux-g++-64
QMAKESPEC is used by qmake. It defines what is the target platform (for example: Windows or Linux) and compiler (for example: Microsoft Visual C++ or g++). Our CSD UG Unix domain machines are "Sun Microsystem Solaris". The installed compiler is g++.

Qt minimal example and compilation

  1. minimal.cpp is a simple Qt program. It simply creates a windows with a label "hello".
  2. To compile it, you need to write a text file as follow:
    # Qt minimal example
    HEADERS +=
    SOURCES += minimal.cpp
    CONFIG += qt warn_on
    #CONFIG += debug
    Text after # is comment and ignore by qmake.
    HEADERS and SOURCES are simply define the header file(s) and source file(s) of your work. We just have one source file here, so we have no header file.
    CONFIG is used to configurate the application. qt option tells qmake to link Qt libraries to the program. warn_on option enables compiler's warning functions. debug option tells qmake to compile the object code and executable code in debug mode.
  3. Save the text file above as our project file minimal.pro.
  4. Type qmake (or qmake -o Makefile minimal.pro) to generate a suitable makefile from the project file.
  5. A file called "Makefile" should be created, or it is updated if it was existed.
  6. Type gmake (or gmake -f Makefile) to compile the code. gmake follows the dependencies rules and their corresponding actions in the Makefile to create an executable code.
  7. A file called "minimal" should be created.
  8. Type ./minimal to run the program.

Building menu and Signals-and-Slots

Files

  1. Download menu.zip and type unzip menu.zip to unpack the file.

Building menu

  1. Change to version1 directory.
  2. In my_widget.h, MyMainWindow is our own class which inherits from QMainWindow. The QMainWindow class provides a main application window, with a menu bar, dock windows (e.g. for toolbars), and a status bar.
  3. To build a menu, you need to know at least two classes: QMenuBar and QPopupMenu. Read the function MyMainWindow::CreateMenu( ) to see how to create menu bar and items.
  4. QMessageBox provides a modal dialog with a short message, an icon, and some buttons.
  5. You need to understand Qt 3.2 Window Geometry in order to locate your UI components on your window.
    [Note: Qt has a more flexible method to arrange UI components called layout manager.]
  6. Compile the program and run it. You should see something similar on the screen:
  7. When you clicked the menu items, they have no response. This is because we have not defined how to handle the click event of the menu items. Qt uses signals-and-slots mechanism to achieve this.

Signals and Slots

  1. In Qt, we have an alternative to the callback technique. We use signals and slots. A signal is emitted when a particular event occurs. A slot is a function that is called in reponse to a particular signal.
  2. There are several version of QPopupMenu::insertItem, you can choose the right one to define a handler of any menu items.
  3. QObject::connect is a static method connects signal from the sender object to member in object receiver, and returns TRUE if the connection succeeds; otherwise returns FALSE.
