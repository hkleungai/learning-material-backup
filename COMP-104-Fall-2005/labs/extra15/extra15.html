Extra 15: The Lord of the Rings with Classes

Re-write Extra14 using dynamic class, and set the number of BlackRiders equal to three times the stage (e.g., on stage 4, there will be 12 BlackRiders). All active units in the game should be modeled as objects.  So, you should use separate classes for Frodo and Black Riders. The current position of  Frodo and Black Riders should be stored in private data members.  Also construct a class BlackRiderArray (see below) that contains a dynamic array of individual Black Riders (BlackRiderArray should have a data member BlackRider *BlackRiderptr). When Frodo reaches a new stage, delete the old Black Rider dynamic array and allocate a new dynamic array whose size is three times the stage number. This will cause the Black Riders to shift to new random locations at the start of each stage with 3*stage Black Riders on the board. Include a copy constructor and destructor in your class BlackRiderArray to avoid dangling pointers and memory leaks.The game board should also be modeled as a class.

class BlackRider{
public:
    BlackRider(); // constructor
    void move(); // BlackRider makes a move
    int getrow() const; // return the row number of BlackRider's position
    int getcol() const; // return the column number of BlackRider's position
private:
    int row;
    int col;
};

class BlackRiderArray{
public:
    BlackRiderArray(); // constructor
    BlackRiderArray(const BlackRiderArray &A); // copy constructor
    ~BlackRiderArray(); // destructor
    void move() const; // all BlackRiders make a move
    bool attackFrodo(Frodo f) const; // check if BlackRider attacks Frodo
    void incrementLevel(); // increase the number of BlackRider when the level is increased by 1
    bool anyBlackRidersHere(int r, int c) const;// check if a BlackRider located at (r,c)

private:
    BlackRider *BlackRiderptr;
    int numBlackRiders;
};
