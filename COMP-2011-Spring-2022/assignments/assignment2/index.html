Problem description

The input is an unique positive integer array (e.g. [1, 2, 3, 4]) and a target integer (e.g., 3). The objective is to build a simple math formula that gives the target with a subset of the given array.

In the formula you can choose to include the positive version (i.e. addition) or the negative version (i.e. subtraction) of each input number, but the relative order of the numbers must be the same as their relative order in the array.

Suppose the input array is [1, 2, 3, 4], let's see some examples to illustrate the rules we follow:

  • "1+2+3+4" is OK.
  • "1+2+4+3" is NOT OK because 4 must appear after 3.
  • "2+4" is OK. (note that we can choose not to include all numbers)
  • "2-4" is OK. (you can do subtractions)
  • "-2-4" is OK.
  • "-1+3-4" is OK.
  • "3" is OK. (a number by itself is also considered as a valid formula)
  • "-3" is OK.
  • "" is NOT OK. (you must have at least one number in the formula)

The solution may not be unique, for example, with array [1, 2, 3, 4], and target 3 the following 5 formulas work:

  • "1+2"
  • "1-2+4"
  • "-1+4"
  • "2-3+4"
  • "3"

Our program will look for the "best" solution. By "best", we will follow a very specific ranking system:

  • We always prefer including an earlier number (either positive or negative version) than not including it. E.g. for array [1, 2, 3, 4] and target 3, "1+2" is better than "3" and "-1+4" is better than "2-3+4".
  • We always prefer including the positive version of the number than the negative version of it. E.g. for array [1, 4, 2] and target 3, "1+2" is better than "-1+4".

In fact, the 5 formulas listed earlier are already ranked from "best" to "worst". More ranking examples will be provided in the section where we describe the solve function later.

Given function

The printArray function has been given to you as a hint on how you may solve the other recursion problems in this assignment.

void printArray(int array[], int arraySize)
This function prints the given array of the given arraySize as a formula. As one of the assignment tasks (you can consider it as task 0), you need to read its code yourself to understand exactly how it works and what it prints. We are not going to explain it for you here.

In terms of the implementation, you will see that it is simply calling the overloaded version of itself (void printArray(int array[], int arraySize, int currentPosition)) with a simple call: printArray(array, arraySize, 0);

The reason for overloading the function and adding the 3rd parameter currentPosition to it is that we want to have a control variable just like what you have in a for-loop (like the i in for(int i=0;i<10;i++)...) You can see in the overloaded function implementation how it is used.

We hope that you can learn from this example to use this approach which will be very useful for the following tasks. While you can create the overloaded versions of any function, you should NOT modify the original function prototype specified in the task description and the skeleton code. That is, for instance, if the task asks you to implement void printArray(int array[], int arraySize) which takes two parameters, you should implement this exact prototype, but please feel free to add the overloaded version of it (e.g. void printArray(int array[], int arraySize, int currentPosition)) which takes additional parameters if needed, like we did for printArray. In the main function, you can see that, we only use the original prototypes to call those functions. For example, for printArray, we only provide two parameters (instead of three parameters) to it in the main function according to its original prototype. If you have modified the original prototype, the testing code in the main function won't be able to call your function for testing.

  • Q: When both of the two rules apply: "prefer including an earlier number than not including it" and " "prefer the positive version over the negative version", how do we decide which formula is ranked higher / better?
  • A: We will basically always look at the first differing element. Take the given test case #7 as an example: input {88, 13, 1, 3, 5, 9} with an target of 9. "13+1-5" is better than "13-1-3" because the first differing element is the second number (i.e. +1 in the first formula and -1 in the second formula). It is "1" for both formulas. As the positive version of it is considered as the better one (i.e. +1 is better than -1), the first formula is considered as overall better.

    On the other hand, for input {1, 2, 3, 4} with an target of 3, "-1+4" is better than "2-3+4" because the first differing element is the first number (i.e -1 in the first formula and 2 in the second formula). They are different numbers. As "1" (which -1 is based on) is an earlier number in the input array than "2", the first formula is considered as overall better.

  • Q: For solve, should we include 0 in the result array?
  • A: No. 0 is not a positive number so it will never be in the input array. To construct the formula, we only use/consider the numbers in the input array.

  • Q: For solve, if the input is {1, 2, 3, 5} and the target is 1, is "1+2+3-5" better than "1"?
  • A: Yes. The former contains “2” while the latter doesn’t, so it is better due to the rule “prefer including an earlier number than not including it”. Here is the list of all solutions ranked from the highest to the lowest for input array {1, 2, 3, 5} and target 1, FYI:
    • 1+2+3-5
    • 1-2-3+5
    • 1

    • -2+3

  • Q: What do you mean by "an earlier number" in the rule "prefer including an earlier number than not including it"?
  • A: An “earlier number” means an “earlier number in the input array”. We really are talking about the number appearance ordering in the input array sequence. You may refer to the follow-up discussion in this post for further explanations and additional examples.
