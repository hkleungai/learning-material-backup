  8. What is the difference between <fstream> and <fstream.h>? Same to <iostream> and <iostream.h>.
    The top of your program should look like this:
    #include <cstdlib>
    #include <iostream>
    #include <ctype.h>
    #include <conio.h>
    #include <time.h>
    #include <fstream>
    #include <string>
    using namespace std;
    Why is <fsteam> ok, and <fstream.h> not ok? It is because there are some incompatibilities between the older libraries (the ones that end in .h, such as fstream.h), and the newer libraries (the ones that don't have the .h, such as fstream). The "using namespace std;" line says to use the newer libraries as default when the .h is not given. (This is specific to VC++, so probably it would work differently in Borland C++.) Libraries are always a bit tricky in C++, since they are not part of the language definition, but are added by the compiler builder (never trust Microsoft to add anything unless their competitors already have it :).
