COMP2011 Lab 4


This lab is prepared for you to practice the 1-D array. In this lab we are going to get familiar with the index, boundary, and value of a 1-D array.

Introduction

In 2022, the snow-white and seven dwarfs move their houses to a community. The 8 houses are circular in the arrangement as the figure illustrated above. One day, a thief sneaked into the community, and wanted to steal as many as he could. However, the community has a security system, which will raise alert when the two stolen houses in the adjacent two nights are too close. In this lab, we will design a safe stealing sequence for the thief to steal but not raise the alert.

In the program, we use a 1-D array to represent the community of arbitrary size, which is set to be 8 for the above example. The index of each element in the array corresponds to the house ID. Firstly, all of the elements in the array are initialized to 0. When a house is stolen, the value of this element should be set to be the same as the stolen night.

Inputs:

  • The first stealing target of the thief.
  • The alert distance of the community's security system.

Rules:

  • Do not raise the alert, for both forward and backward directions. For example, when the alert distance is 2, and the thiefâ€™s current position is 3, then the next target ID must be equal or greater than 5 (>= 3 + 2), and equal or less than 1 (<= 3 - 2).
  • You only need to consider the distance between the next target with the current position, not the previous stolen houses.
  • The thief cannot choose the house which has been stolen. And the closer the next target is, the better. For example, in this situation | 0 | * | 0 | 1 | 0 | 0 | 0 | 0 | (* means the thief's position and 0/1 means the state of the house) and the alert distance is 2, the next target should be house 4, not 3, not 5 neither.
  • The thief's stealing direction is to the right (clockwise in circular arrangement). When the thief is in house 6, his next target is house 0, 1, 2....
  • The thief should leave immediately(finish the program) when no valid house that satisfies the first three rules.

Outputs:

  • The states of the houses in each night.

Tasks

In this lab you are given the code_skeleton, and you need to finish one while loop in the main( ) and two functions:

  • Task 1: while loop in the main( ).
    In each loop, you need to update variable 'night', call 'steal_house( )' function to get the thief's next target and update the thief's position, and call 'print_houses( )' function to print the states of the houses. When the next target is invalid, the thief should break out the while loop.
  • Task 2: steal_house().
    In this function, the inputs are the array, thief's current position, , the alert distance, and the 'night'; the output is the next target. When the next target is valid, return the target index, else return -1. You also need to modify the value of the target element to 'night'.
  • Task 3: print_houses( ).
    In this function, the inputs are the array, the thief's current position, and the 'night'. The thief uses '*', valid houses use '0', and stolen houses use '1, 2, 3, ...' (stolen night).

Grading Scheme

Each lab carries 3 points in total.

  • By attending a lab, a student gets 1 point for attendnace (but only if it is clear that the student does put in some effort).
  • By completing the lab exercise correctly and answering TAs' question (if there is a question) correctly, the student gets the remaining 2 more points.
If a student is unable to finish the lab and demonstrate to the TA by the end of the lab, they can only get the attendance point.
