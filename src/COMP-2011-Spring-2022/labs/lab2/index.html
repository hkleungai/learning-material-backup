COMP2011 Lab 2


Introduction

In this lab, you are going to implement the Sliding Puzzle Game. You can try to play the game and get familiar with it here. We provide a skeleton code for you to start this lab.

Game Description

Sliding Puzzle Game is made up as a set of blocks which are placed inside a frame. Within that frame the blocks can be moved around only by sliding. To solve the puzzle, you need to slide the blocks near the empty position and make sure that all the blocks are in their correct positions.

In this lab, you need to implement a Sliding Puzzle Game with 2x2 frame, which has 3 blocks "A", "B", and "C". The fourth block is an empty block denoted as "*".

Meanwhile, the positions in this 2x2 frame are noted as 1, 2, 3, 4, where position 1 is in the top left corner, position 2 is in the top right corner, position 3 is in the bottom left corner, and position 4 is in the bottom right corner.

At the begining, the puzzle is initialized as following: 

Position info:
12
34
Puzzle status:
BC
A*

To solve the puzzle, the final status should be: 

Position info:
12
34
Puzzle status:
AB
C*

Implementation

  • At the start of each round, print out the round number, position info, and the puzzle status.
  • In each round, the program should prompt the player to select a position number for sliding. You must perform input validation:
    • The position number must be 1, 2, 3 or 4.
    You should ask the player to input again if his input is incorrect. Note:
    • The input must be the position number (1 to 4), not the character "A", "B", or "C".
    • You don't need to consider the situation that the chosen block is not avaliable to slide.
  • Then, you need to swap the location of empty block "*" and the block at chosen position.
  • Check whether the puzzle is solved after this sliding.
    • If no, the game goes to the next round.
    • If yes, the game is over. You need to output the total steps and the final puzzle status.
