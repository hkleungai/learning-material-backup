o    06-03-2015： PA to test equal number of a’s and b’s

  • In the class today, I raised the question about whether we can replace ‘c’ with ‘e’ in the solution. The problem is discussed  in
    http://www.cs.utexas.edu/users/cline/ear/automata/CS341-Fall-2004-Packet/2-Homework/Home13PDAs.pdf

The point seems to be this: If we replace (q, a, c)->(q, ac) with (q, a, e)->(q, c), we might push a even if b is on top of stack.  However, I do not see why it is problem because of non-determinism: a word is accepted if there is one way to go from the initial configuration to (f, e, e).

I believe ‘c’ is introduced to make the machine easy to understand. Although it is correct without ‘c”, it would be more difficult to comprehend.
