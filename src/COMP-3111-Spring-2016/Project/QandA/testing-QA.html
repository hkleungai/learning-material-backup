Q: In the peer acceptance tests, can we assume that the data of the account, client and security holdings given in previous labs exists in the other team's database or only the acceptance testing data given in the Project Resources module?
A: The requirement is that only the acceptance testing data given in the Project Resources module exists in the database.

Posted Apr. 17

Q: For the testing data for peer acceptance testing, is only the data for client Yuki Yuen added to the database before testing? No other testing data created by us is allowed to be tested, right?
A: The peer acceptance test cases can only assume that the data for client Yuki Yuen and her security holdings exist in the securities portfolio management system's database. If there are test cases for other clients or other security holdings, then that data must be added by other acceptance tests.

Q: What is the account number for client Yuki Yuen?
A: The team whose system is being tested should know this and they must be able to tell you the account number if one of your test cases requires it.

Q: Can only the logins yukiyuen and employee be used in the test cases?
A: These two as well as the logins of any other clients that your test cases create.

Q: Can we construct test cases similar to those used in the Activity 1 testing?
A: Yes.

Q: Is it allowed to change the rate of a currency to 0?
A: It is not allowed to change the rate of a currency to 0.

Posted Apr. 23

Q: In the provided testing data we are required to input the information for the client Yuki Yuen. As for the balance of the client, should we first set her balance to be the initial balance and and then buy the given securities or should we set her balance to be the initial balance and input all the security holdings for her without changing her balance?
A: Set the initial balance and then buy the securities using your system's buying function.

Q: During the testing session can the currency rates be changed or they will be the same as the ones in the current external system.
A: They can be changed using the control center of the external system.

Q: Can we change the price of a security during peer acceptance testing?
A: You can use the control center of the external system to freely change the price of any security during testing. This will not count as a test case. Moreover, the price of a security is changed automatically to the purchase/sale price whenever there is a purchase/sale of the security.

Q: In peer acceptance testing, if the first test case, such as create account which is expected to be valid, fails and causes the following test cases, such as buying and selling, to not be able to be performed, which team will get the score of the test cases that cannot be performed?
A: Neither team. You should have a backup plan for testing in case something like this happens (i.e., use the client given in the testing data instead). You should not make too many test cases dependent on each other in case a situation like this occurs.

Q: If we want to test the profit/loss tracking functionality, we will need to know the difference between the current share pice and the past share price. However, since our data is static, how can we determine if the share price has gone up or down?
A: The data is not static. The price of a security can go up or down and can be changed any time using the control center of the external system. So, you can buy a security at a certain price one minute and then change the price of the security immediately and see whether you have a profit or a loss.
