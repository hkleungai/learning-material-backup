DEVELOPMENT TOOLS

The development tools for the course project are Visual Studio, ASP.NET, Git, NUnit and Rational Modeler. The database system is Microsoft SQL Server. These development tools will be explained in the tutorials and used in the labs. You will also be provided with tutorial and lab notes that describe the features and functionality of these development tools.


TIME SCHEDULING

Time is one of the critical resources that you need to manage well in any software development project. While software development is very time consuming and often unpredictable, good planning and time management can go a long way to determining the success of your project. Therefore, do not leave working on a project activity until the last week, last few days or last few hours before its deadline. If you do so, you are just asking for trouble. You have been warned!


INDIVIDUAL CONTRIBUTION OF TEAM MEMBERS

The individual contribution (i.e., effort) of each team member to the completion of team project activities will be used in determining the activity project grade for each student.

The effort of each member of a project team for team project activities will be indicated by completing a table like that shown below. The value for % effort for each team member should be a number between 0 and 100. The total should be less than or equal to (#-team-members*100) (i.e., for 4 team members the total should be less than or equal to 400).

Team member name % effort
   
   
   
   
Total  

The % effort values will be used to adjust the mark of certain project activities for each team member by dividing the % effort values by 100 and using the result as a multiplier to arrive at a final grade for the project activity. If the % effort for all team members is 100 for a project activity, then the multiplier will be 1 for all team members (i.e., there will be no adjustment of the project activity grade). However, if it is less than 100, then the multiplier will be less than 1. If there is disagreement among the team members about the contribution of any team member, the course instructor reserves the right to resolve any such disagreement.

Note:

The maximum % effort that can be assigned to a team member is 100. The minimum is 0.  If the contribution of a team member is below 80%, then an accompanying explanation for why the contribution is below 80% and a meeting of the team member and project manager with the course instructor may be required.

Here is an example % effort assignment for all team members and the resulting multiplier for a project activity mark:

Team member name % effort Multiplier
Andy 90 0.9
Mike 60 0.6
Shirley 100 1.0
Tony 100 1.0
Total 350
