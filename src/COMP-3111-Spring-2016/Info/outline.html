COMP 3111: Software Engineering

Course Outline

OVERVIEW

Software engineering is mainly concerned with following a disciplined approach to software development while making use of appropriate software development tools and techniques. This course introduces the topic through lectures and by giving you a chance to participate in the design, implementation and management of a software development project using industry standard software development tools. Since modeling is an important first step in developing a software system, we first consider how to model software systems using the Unified Modeling Language (UML). We then survey and critique various software development processes and focus on software development using SCRUM, an agile software development process. Next, using the Unified Process (UP) life cycle as a framework, we delve in-depth into the major engineering activities of the software development life cycle—requirements capture, analysis and design, implementation and testing—and show how the UML can be used for modeling and documenting a software system. We conclude with an overview of the major management activities of the software development life cycle—software quality assurance and project management.

OBJECTIVES

This course aims to give students both a theoretical and a practical foundation in the concepts and practices of software engineering. In the theoretical part, students will learn about the principles and methods of current software engineering practices focusing on the use of the UML in modeling software systems. In the practical part, students will become familiar with the disciplined development of a software product under a strict schedule using industry standard tools. After completing the course, students will have acquired the following.

  • An understanding of the concepts as well as practices of software engineering.
  • Experience with analyzing, designing, implementing and testing a software system.
  • Hands-on experience with an industry standard integrated development environment (IDE).

SOME QUESTIONS THE COURSE WILL TRY TO ANSWER

How can we capture and document what the client would like the system to do?
How can we transform the client's requirements into a software system?
How can we build confidence that the system is implemented correctly and behaves as desired?
How do we organize the development project so that we meet our budget constraints and deliver a system on time?
