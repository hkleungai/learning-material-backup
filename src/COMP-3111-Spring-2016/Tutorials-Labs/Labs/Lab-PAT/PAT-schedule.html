Activity 3: Peer Acceptance Testing Schedule

IMPORTANT TESTING INSTRUCTIONS

Your team members must arrive at Lab 4 (Room 4210) at least 10 minutes before the time specified for the team in the schedule and be prepared to start the testing at the start of the lab session. Two members of the team whose system is being tested will run the test cases that are provided by another team on the their own system while a third member of the team whose system is being tested will participate in running the test cases constructed by the team whose system is being tested on another team’s system.

  • The code for a team's system must run from a folder on the desktop.
  • Test case test data that requires extensive typing must be available in a file that can either be downloaded or is available on a USB drive so that it can be copied and pasted.
  • The ExternalDatabases.mdf file in the App_Data folder must be the one available from the ADONETLabActivity-Download folder from Week 4 of the Tutorial and Lab Schedule module of the course web page.
  • The Peer Acceptance Testing data available from the Project Resources module of the course web page must already be entered into a team's database and must not have been used for testing. If this data is not in a team's database or it is found to not be exactly as specified, the team's testing session will be canceled and the team will get zero for Peer Acceptance Testing.
  • Your web site also must have the following employee account available for use during the testing session: user name "employee"; password "employee##".

If the members of a team fail to show up at the specified time, no other time will be arranged for that team's testing and the team will get zero for Peer Acceptance Testing.

During a testing session, only members of team's whose system is being tested are allowed to be in Lab 4. If a member of any team whose system is not being tested is found to be in Lab 4 during a testing session, then that team's testing session will be canceled and the team will get zero for Peer Acceptance Testing.
