Part 1 - The SVG Elements (30 marks)

  • The sky
    • The sky is a <rect> with a gradient - 5 marks
  • The ground
    • The ground is a <rect> with a gradient - 5 marks
  • The cloud
    • An appropriate SVG image is used - 5 marks
    • There are at least two clouds in the game - 5 marks
  • The player
    • An appropriate SVG image is used - 5 marks
  • The obstacle
    • An appropriate SVG image is used - 5 marks
  • Note that all SVG elements must be significantly different to the ones given in the lab page, e.g. different colour, shapes and so on

Part 2 - The CSS Animations (30 marks)

  • The CSS animation of the clouds
    • The clouds move from right to left (or left to right) of the screen - 5 marks
    • Each cloud starts its animation at a different time - 5 marks
  • The CSS animation of the obstacle
    • The obstacle moves from right to left of the screen - 5 marks
    • The obstacle is initially hidden - 5 marks
  • The CSS animation of the player
    • The player jumps by moving up and down - 5 marks
    • The player does not initially use the animation - 5 marks

Part 3 - Controlling the Animations (40 marks)

  • Moving the obstacle
    • The obstacle shows up at random time and moves across the screen repeatedly - 5 marks
    • The obstacle is shown and hidden appropriately during the game - 5 marks
  • Making the player jump
    • The player jumps every time the spacebar is pressed - 5 marks
  • Game over
    • Appropriate condition is used for the game over situation, i.e. the player is near the ground and the obstacle is close to the player - 10 marks
    • All animations, excluding the game over text, are stopped when the game is over - 5 marks
    • The game over text is shown when the game is over - 5 marks
    • Some kind of animation is used when the game over text is shown - 5 marks
