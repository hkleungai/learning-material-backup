COMP2011 Lab 10: Final Exam Scheduling Using C++ Class

Introduction

The final week is coming. Our university staffs are working hard to schedule the final exam of many courses. In this lab, suppose you are required to help the staffs implement a simple exam scheduling program using the C++ Class. A skeleton code has been provided to help you get started.

Consider the following application scenario. A set of courses apply to use a classroom as the place of their final exam. One can define a Course class and a Classroom class. See Course.h and Classroom.h of the skeleton code for their definitions.

Concretely, a Course class has the following data members:

  • char* course_ID: course ID, e.g., COMP2011
  • char* course_name: course name, e.g., Programming_with_C++
  • int start_time: expected start timestamp of the final exam
  • int end_time: expected end timestamp of the final exam
  • int num_students: number of students that will attend the final exam of this course
For simplicity, we use an integer timestamp (e.g., 0, 10, 100, etc.) to represent the start_time and end_time instead of specifying the day, hour, and minute.

A Classroom class has the following data members:

  • char* room_ID: classroom ID, e.g., RM2464
  • int quota: number of seats, i.e., the maximum number of students of an exam
  • Course* app_list: a Course array records all the courses applying to use this classroom
  • Course* exam_list: a Course array records the courses that can use this classroom as their final exam place
  • int num_app: number of courses applying to use this classroom, i.e., length of app_list
  • int num_exams: number of exams that will be held in this classroom, i.e., length of exam_list
In addition to the accessors and mutators, the Classroom class also has the following member functions:
  • void receive_app(Course course): receive the application of a course, where we only receive the application of courses whose number of students is less than or equal to the classroom's quota
  • void gen_timetable(): generate the timetable of final week of this classroom
  • void print_timetable(): print the timetable of final week of this classroom
In this lab, we consider a simple case that only one classroom receives the applications from multiple courses. Note that there may be some time conflicts among the courses. Your program should decide whether to accept the application of a course such that
  • (1) the number of students of each course should be less than or equal to the quota
  • (2) this classroom can hold as many examinations as possible
  • (3) there are no time conflicts among the exams
Such a task can be formulated as the Interval Scheduling Problem. You are required to implement a greedy algorithm, which can ensure the number of exams that can be held in a classroom is maximized.

Tasks

  • Task 1: Implement the mutator of the Course class (see Course.cpp of the skeleton code).
  • Task 2: Implement the constructor of the Classroom class (see Classroom.cpp of the skeleton code).
  • Task 3: Implement the member function receive_app of the Classroom class (see Classroom.cpp of the skeleton code). You are required to add a course to app_list (if its number of students is less than or equal to the classroom's quota) and increment num_app.
  • Task 4: Implement the member function gen_timetable of the Classroom class (see Classroom.cpp of the skeleton code). You are required to implement a greedy algorithm of the interval scheduling problem including the following 2 steps.
    Step 1: Sort the app_list according to the course end_time;
    Step 2: From the beginning of app_list, if a course has no time conflicts with those courses already added in exam_list, add it to exam_list and increment num_exams.

Some information of the classroom and courses should be saved in a file with the following format. The first line gives the (1) course ID, (2) quota, and (3) number of courses (applying to use this classroom) separated by whitespaces. Starting from the second line, each line gives the information of a course, including (1) course ID, (2) course name, (3) expected start timestamp, (4) expected end timestamp, and (5) number of students, separated by whitespaces. For each course, ensure that the end timestamp is larger than the start timestamp and there are no whitespaces in the course name (i.e., use 'Programming_with_C++' instead of 'Programming with C++'). We assume that there are at most 20 courses applying to use a classroom. We also assume that the string length of the classroom ID and course ID should be less than 20. The string length of course name should be less than 100. An example of the input file is as follow.

RM2464 300 3
COMP1021 Introduction_to_Computer_Science 10 160 250
COMP2011 Programming_with_C++ 155 255 200
COMP2611 Computer_Organization 0 150 103

You can compile the program using the following command.

  • For Windows: g++ -o lab10.exe Course.cpp Classroom.cpp main.cpp
  • For Mac: g++ -o lab10.out Course.cpp Classroom.cpp main.cpp

Grading Scheme

Each lab carries 3 points in total.

  • By attending a lab, the student gets 1 point for attendance (but only if it is clear that the student does put in some effort).
  • By completing the lab exercise correctly and answering TAs' question (if there is a question) correctly, the student gets the remaining 2 more points.
  • You are recommended to turn on the camera while demonstrating.
If the student is unable to finish the lab and demonstrate to the TA by the end of the lab, they can only get the attendance point.
