COMP1022Q Lab 6 'Escape the Monster' Game

Marking Scheme

  1. Handling the player's position
    • Create shapes for the reset button and the arrows (5 marks)
    • Complete MovePlayerLeft subroutine (5 marks)
    • Complete MovePlayerRight subroutine (5 marks)
    • Complete MovePlayerUp subroutine (5 marks)
    • Complete MovePlayerDown subroutine (5 marks)
  2. Handling the monster's position
    • Check if the game is being played (5 marks)
    • Hide the current position of the monster (5 marks)
    • Change the row of the monster to move towards the player (5 marks)
    • Go back to the previous row if the newly selected row is not white (5 marks)
    • Change the column of the monster to move towards the player (5 marks)
    • Go back to the previous column if the newly selected column is not white (5 marks)
    • Set the new monster's postion with new row and new column, and change the cell color to green (5 marks)
  3. Checking for the end of the game
    • Check if the game is being played (5 marks)
    • Check if the player has reached the right side (5 marks)
    • Based on the difficulty value, check whether the monster has eaten the player (5 marks)
  4. Diagonal movement arrows for the player
    • 4 extra arrow shapes are in the appropriate positions on the worksheet (5 marks)
    • 4 extra subroutines are implemented to support diagonal movements (5 marks)
    • All 4 new subroutines correctly handle the game boundaries (5 marks)
      • The user cannot move out of the game area
      • No error can be generated when moving
  5. Difficulty level
    • Write a new function Function ... End Function which returns the distance between the monster and the player (5 marks)
    • The distance value should be clearly displayed in a cell (5 marks)
  6. Enter your name and student ID number in Row 24.
