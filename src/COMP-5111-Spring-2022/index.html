Course Description

This is a postgraduate core course counted towards the area of software technology. The goal of this course is to introduce the principles of various automated analysis and testing techniques as well as the ways that they can be used to manage software code quality. Students will acquire fundamental knowledge of program abstraction, verification, testing, coverage, analysis, reliability, and fault detection. The course will also discuss how to carry out the empirical experimentation for program analysis and testing. Wherever applicable, concepts will be complemented by tools developed in academia and industry. This enables students to understand the maturity and limitations of various techniques discussed in the course and put them in practice. The course requires prior programming knowledge in Java.

Course Learning Outcomes

The intent of this course is to provide students with the fundamentals of software analysis and testing techniques. They will learn these techniques from two perspectives: concepts and tool support. Topics will be explored at different levels of depths. Following the first perspective, students will study the quality issues arising from software maintenance and the theories to address them. Following the second perspective, students will study how these techniques are supported by tools and their underlying engineering issues. Many of these tools are adopted by the software community to assure code quality.

Upon completion of this course, students will be able to apply the fundamental knowledge of fault analysis, test coverage, program instrumentation, test generation, fault localization, mutation analysis, program repair, pointer analysis, program abstraction, program repair, mining software repositories and vulnerability analysis. Students will also be able to point out the maturity and limitations of various analysis and testing techniques. This course assumes prior knowledge of Java Programming.
