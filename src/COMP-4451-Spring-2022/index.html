Assignments

All assignments are related to a course open-ended game project. The project can be done individually or in pairs. Pairs are encouraged (only 50% extra work expected). The goals and expectations of the project will be discussed in class. In short, you should strive to develop something that considers:

  • Creativity: New and different, not present in existing games.
  • Challenges: It should involves challenging programming in at least one aspect of the game. Using game engines is allowed if it helps you in some aspects, but it is not encouraged and you will be graded on the complexity of the parts that you implement yourself.
  • Playability: It should be a complete game that is playable, that is addictive, appealing, or pleasurable to play multiple times. The game mechanics should have a good balance of elements, goals, and rewards to keep players entertained.

Here are a couple of examples of previous games, based on request. Please do not constrain too much on what you see here, both in style or formatting of the game. It is mainly to see the challenges and level of difficulty of some interesting or creative games: MP4 with both videos.

A0 Group formation
Deadline: Wednesday, Feb 16, 2022 at 23:59 (Canvas submission).

Submit the name and email of your group mate under the "Group formation" Canvas assignment. For example, if your group mate is John Smith, your should submit "John Smith, johnsemail@connect.ust.hk". If you are doing an individual project, submit the text string "None". We will use this information to assign Group IDs which you will then specify in your later project submissions.

A1 Game proposal
Deadline: Wednesday, Feb 23, 2022 at 23:59 (Canvas submission).

Components:

  • Proposal document: Submit a proposal.pdf file per group under the "Game proposal document" Canvas assignment. The proposal should use this template file, exported in PDF format before submission. Refer to the template for detailed guidelines.

  • Pitch video: Submit a proposal.mp4 file per group under the "Game pitch video" Canvas assignment. It should consist of an overview of your planned game. You should first present the motivation and idea behind your game, what inspired it and specifically how it differs from existing games. Then you should put it in the context of the different game categories, structures, and other characteristics, and discuss how you plan to use and balance different mechanics of gameplay. Finally, you should go more in depth about the challenging programming-related aspects of the game. This can be related to any aspect you want to explore more deeply, such as rendering, modeling, physics, AI, networking, etc... The length of the video must be between 3m30s and 4m. This is a hard time constraint that will be enforced during grading. Important: Refer to the Video recording details below for additional instructions.
A2 Game progress
Deadline: Wednesday, Apr 6, 2022 at 23:59 (Canvas submission).

Submit a progress.mp4 file per group under the "Game progress" Canvas assignment. You should start with a brief 1-minute summary of your game to refresh the audience's memory. Then, the most important part of this video progress report is to show a demo of how your game looks at the moment. You are not expected to have more than a very basic gameplay prototype with the high level logic and preliminary game play. It could be missing many of the important elements which you can then refine over the remaining weeks. In addition, your presentation should clearly discuss all the programming-related aspects that you have completed so far as well as the ones you plan to do for the remaining weeks (note that this part should not include direct game engine feature usage, but rather the code/scripts for the game parts and different algorithms that you implemented and is custom for your game). This part could include details of your code related to rendering techniques, physics, AI, etc... The length of the video must be between 3m30s and 4m (same as proposal). This is a hard time constraint that will be enforced during grading. Important: Refer to the Video recording details below for additional instructions.

A3 Game submission
Deadline: Wednesday, May 4, 2022 at 23:59 (Canvas submission).

Components:

  • Code: Submit a code.zip file per group under the "Game code" Canvas assignment. If the project is not WebGL-based and requires compilation to generate an executable, please also include the executable. The root directory of your zip file should contain a readme.txt file including the group ID and members' names and emails, and instructions on how to run your game.

  • Presentation video: Submit a presentation.mp4 file per group with your rendering project presentation under the "Game presentation video" Canvas assignment. You should start with a brief 1-minute summary voice-over of your game to refresh the audience's memory. In this summary you should show a short clip of your game that best demonstrates its features, gameplay, and complexity. Then the video should go into more in-depth detail of your game, covering all the features implemented, the different levels if applicable, the UI interface, and any other aspects that you spent time developing. Finally, the video should end with a set of slides that outlines all of the challenges faced, both in terms of design and engine usage, as well as programming. Be very thorough and specific about which programming challenges that were implemented from scratch as well as those that were taken or extended from existing libraries. The length of the video must be between 3m30s and 4m (same as previous ones). This is a hard time constraint that will be enforced during grading. Important: Refer to the Video recording details below for additional instructions.
A4 Game report
Deadline: Wednesday, May 11, 2022 at 23:59 (Canvas submission).

Submit a report.pdf file per group under the "Game report" Canvas assignment. The report should use this template file, exported in PDF format before submission. Refer to the template for detailed guidelines.

Video recording details: Your videos must start with a slide showing the game name, group ID, and member names and emails for at least 3 seconds. The speaker's face should appear at the corner of the video during the narration. For groups of two members, each member must present at least part of at least one of the videos. The content shown should be nicely formatted to consist of slides and videos/demos. Please do not simply scroll through the submitted documents and code files with narration. The videos should be in mp4 format at FullHD 1920x1080 resolution. This can be accomplished by recording the videos within a Zoom meeting with screen sharing showing slides/demo and speaker video turned on. The recorded meeting mp4 files can then be trimmed to the appropriate range, downloaded in FullHD, and submitted to Canvas. Please make sure to adhere to the time constraints and you cannot accelerate the voice narration using software since it sounds very artificial. Experiment with a video recording approach well before the deadline so as to avoid last minute surprises.

Late Submission Penalty: All videos must be submitted in time to be shown in the following day's class, otherwise no credit will be given for those components. Please plan ahead of time. For the other components, there is a penalty of 20% if submitted up to one day late and 40% for up to three days late. There will be no credit for submissions that are submitted beyond that time frame without prior arrangement.
