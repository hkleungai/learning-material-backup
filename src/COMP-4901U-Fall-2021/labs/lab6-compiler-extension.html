Lab 6

Compiler Extension for the Amy Language (Optional)

COMP 4901U Home COMP 4901U on Canvas

You have now written a compiler for Amy, a simple functional language. The final lab project is to design and implement a new functionality of your own choice on top of the compiler you built so far. In preparation for this, you should aim to learn about the problem domain by searching the appropriate literature. The project includes:

  • designing and implementing the new functionality

  • documenting the results in a written report document

This project is optional, meaning that you do not have to complete it.

Grading scheme: If you choose to complete this project, the marks you obtain for it can be used to recover at most half of the marks you lost in the previous labs/mini-projects. For example, if you have an average of 40% in your mini-project grades, then obtaining 100% in the optional project will prop your overall labs grade to 40 + 100/2 = 90%. The overall labs grade is capped at 100%. So if, on the other hand, you had an average of 60% in your mini-projects and you still obtained 100% in the optional project, your new grade will be 100%, not 110%.

Because this is potentially quite a big bonus, the compiler extension project will be evaluated rather strictly (do not expect an easy 100% grade).

Selecting a Project Topic

In the following document, we list several project ideas, but you should also feel free to submit your own.

However, you should make sure to contact me (the course instructor) before selecting a project, even if it is already a listed project. Indeed, I will try to make sure that no two groups work on the same project topic.

Because not all projects are equally difficult, we annotated each of them with the expected workload. The suggested projects cover a wide range of complexity, and we will evaluate your submissions with that complexity in mind. For instance, for a project marked with (1) (relatively low complexity) we will be expecting a polished, well-tested and well-documented extension, whereas projects on the other end ((3)) may be more prototypical. For all submissions, however, we require that you deliver code that compiles and a set of example input files that demonstrate the new functionality.

Project ideas

Project Orientation

To give you a chance to validate your understanding of the project and what’s expected of you, please feel free to contact me by email and possibly set up a meeting in person. But before you do, think about the following questions:

  • What are the features you will add to the compiler/language?

  • What would be some (short) programs highlighting the use of these features?

  • What changes might be required in each compiler phase and/or what new phases would you add? (Very roughly)

Project Implementation and Report

Deadline: December 18, 2021, 23:00

Your implementation and a report are due on this date at the latest. Any later submission will not be considered.

Note that you should try to submit your work earlier if possible, which will allow me to come back to you with feedback and potentially suggest improvements or fixes. If you submit right before the deadline, I will not have any time to come back to you because the final grades will be due for reporting to the CSE department soon after.

You will develop your project on top of your implementation of Amy. Please submit all the source files in your project to me, in a zip archive, so I can properly test it. Your zip file should contain:

  • Your implementation, which must, to be graded at all, compile and be able to run non-trivial examples.

  • A subdirectory extension-examples/ which includes some examples that demonstrate your compiler extension in action.

  • A subdirectory report/ which includes a PDF summarizing your extension.

If you did not manage to complete your planned features, or they are partially implemented, make this clear in your report!

You are encouraged to use the following (LaTeX) template for your report:

  • LaTeX sources

A compiled PDF version of the template with the required sections is available here:

  • PDF Example

Although you are not required to use the above template, your report must contain at least the sections described in it with the appropriate information. Note that writing this report will take some time, and you should not do it at the last minute. The final report is an important part of the compiler project. If you have questions about the template or the contents of the report, make sure to ask them early.

Please try to stay within 6 pages or less for your report, but do include all code examples and technical descriptions needed to truly understand your contribution. A concise but well-written report is preferable to a long but poorly-written one.
