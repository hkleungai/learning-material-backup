Proper behavior in Unix lab

  1. NEVER reboot the workstation. NEVER. You'll blow up other's people work if you do so, and you can also crash the file system (i.e. hard disk). Talk to the TA or send an email to system staff cssystem@cs.ust.hk for help if you have trouble with a workstation.
  2. Do not left the computer you've login unattended. Otherwise, other people can access your files. It has happened before that someone has login to a computer and then left it unattended. Later he found out that his solution for a programming assignment was sent out as a broadcast email to everybody in the class.
  3. If you really need to leave for a short time, say, going to the washroom, you can lock the workstation by the command xlock. However, it is extremely selfish and inconsiderate if you lock your workstation and then leave for a long time, say, going to have lunch/tea/dinner, or attending lectures/tutorials. If you find anyone committing this offence, report to cssystem@cs.ust.hk.
  4. Note that you need to keep the door of lab 2 closed. If it is left open for more than 30 seconds, it'll beep.

Basic ideas on Unix

  1. Unix is a command-line driven operating system. This is quite different from windows, where you do many tasks by clicking and dragging.
  2. To get a new command window, type xterm & in cmdtool/console/terminal.
  3. To start Netscape, type netscape &
  4. The symbol "&" means "background". For the time being, just remember that whenever your command results in creating a new window, add the symbol "&" at the end.
  5. Unix is a multi-user operating system. This means that all users can use the machine that you are using now. In addition, other users may be able to read your files if you do not protect them.
  6. The initial directory (folder) you are at when you login is called "home directory". Its pathname is "/homes/<your username>" and it is also represented by the symbol "~".
  7. In your home directory there should be a directory called "DOS". The files that you placed in the network drive in the Windows environment for CSD are located there. Do not touch it unless you know what you are doing.
  8. Try the command ls -ld ~ Other people cannot read your file if you get something started by "drwx------ ", such as

        drwx------ 6 martin 512 Feb 5 15:52 /homes/martin

Common trouble for beginners of Unix

  1. Unix is case-sensitive. This means that "A" and "a" are different. Most Unix commands consist of lower-case letters only. If your password consists of both upper-case and lower-case letters (which is a good practice), make sure that you type your password in the correct case. Also, when you are asked to create a directory "comp151" under your home directory, do NOT create COMP151. They are different.
  2. Sometimes your terminal may set suddenly locked up. Try press ctrl-q, and this may unfreeze the terminal. If this is the case, you have just accidentally pressed ctrl-s, which locks the terminal.
  3. Unix do not have a recycle bin. Everything you delete or overwrite is immediately lost. To give yourself some protection, you can add the command alias rm rm -i; alias cp cp -i and then put this into a file .cshrc_user under your home directory.
  4. "Space" is usually significant in UNIX. When there is no space, do not add one. When there is one, do not forget them. Also "tab" is different from white space! Newline cannot be added arbitrary, too!
  5. Many symbols in Unix have special meaning. Some of them look very alike but are  different. For example, there are three types of quote: forward quote `, backward quote ' and double quote ". There are also two types of slash: slash / and backslash \. Be careful on this!
  6. You have a disk quota of 8M. Use quota -v to see how much space you have. When you are full of disc quota, the system may response weirdly.
  7. To print in the CSD, use the command lpr. You can check your print quota in CSD by ckpq. 

Some Unix tips

Some nice features in tcsh

You can use "up-arrow" and "down-arrow" to scroll through previous commands. If this doesn't work, ctrl-p and ctrl-n can be used instead.
Ctrl-a brings you to the start of a line. Ctrl-e brings you to the end.
When you want to type in the name of a file, you may not need to type in all the characters. Press <tab>, and the system can complete the filename as much as it can. In case of ambiguity, you can type ctrl-d to see the list of potential filenames. This also works for command, too.
If you want to repeat the latest command that begins with l, you can type !l as a shortcut.

Some nice features in xterm

You can change the font size by 
  1. Let your mouse point to somewhere in the xterm window.
  2. Press the control key and hold it.
  3. Press the rightmost mouse bottom.
  4. Then a menu of font size appears. Change to huge if you like large fonts.
Note that there are other types of menus if you press the leftmost or middle button while holding the control key. You can try them yourself.
Copy-and-paste in xterm
  1. Use the leftmost button to select the text you want to copy.
  2. Move the cursor (usually NOT by mouse) to the position you want to paste.
  3. Press the middle mouse button

Working remotely

You can also work remotely by using SSH clients under Microsoft Windows. Just enter "csl2wk??.cs.ust.hk" when you are prompted for a hostname, where ?? stands for a number from 1 to 40. You can also work at home this way. Note that this is the recommended way to work at home: if you do your assignments using Visual C++, you are at your own risk!!

using SSH

The most basic data files which can easily be accessed remotely are the files stored on UNIX file servers. E.g. User home directory, project directory. Users can SSH to anyone of the departmental UNIX workstations and access the files directly or easily copy the files around.
